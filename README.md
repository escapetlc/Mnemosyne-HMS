# 谟涅HMS — 轻量记忆系统

> **Mnemosyne Loci Holographic Memory System** — 模型记忆提取、认知维护与跨会话记忆恢复  
> 零基础设施，纯文本协议，复制粘贴到任意 AI 窗口即可执行。  
> 不是日常工具，是窗口溢出/限额耗尽/对话中断时的急救与重建工具。

---

## English Overview

Mnemosyne HMS is a lightweight, zero-infrastructure memory system designed for AI conversational windows. It converts long dialogues into structured memory cards (JSON/Markdown) that can be manually restored in any new chat window.

- **No server. No database. No API keys. No code.**  
- Just copy, paste, and let any AI model follow the protocol.  
- Bilingual fact-type labels (`world(世界)` / `opinion(观点)` / `observation(观察)` / `decision(决策)` / `pending(待办)` / `hazard(警示)`) eliminate cross-model semantic drift.  
- Inspired by the [HMS (Shadow-Weave) academic project](https://github.com/Shadow-Weave/HMS), adapted to pure-text protocols.  

The repository is primarily in Chinese. Core protocol terms are bilingual. For protocol details see `docs/系统架构.md`.

你跟AI聊了整个下午。项目思路理清了，方案框架搭好了，它甚至记住了你的写作风格和你讨厌的表达方式。然后——对话窗口溢出，或者平台限额耗尽了，或者你关掉了它。

第二天打开新窗口，它看你就像看陌生人。

你得从头解释你是谁、你在做什么、之前讨论到了哪一步。它很聪明，但它什么都不记得。就像每天早上失忆的同事——能力很强，但每天你都得重新做一遍自我介绍。

谟涅HMS 就是为了解决这个问题而生的。不过它不是让你每天都用的东西，而是当**窗口满了、限额超了、对话断了、不得不重建时**——用来抢救和重建的急救包。

谟涅HMS，全名叫"谟涅轻量记忆系统"。名字听着大，拆开看很简单。

谟涅（Mnemosyne）是希腊神话里的记忆女神。古希腊人相信，把知识交给谟涅女神保管，她就不会让你忘掉。系统用这个名字，做的事情跟女神一样：在你关掉窗口之后，替你把东西存下来。

HMS 来自 Shadow-Weave 团队做的学术研究项目，全称是 Holographic Memory System（全息记忆系统）。项目提出了完整的方法：怎么从对话里提取信息、怎么把信息组织起来、怎么在需要的时候找回来。但它需要服务器、数据库、API 接口——得搭整套基础设施才能跑起来。

谟涅HMS 做的事情，是把 HMS 同样的设计思想搬到了纯文本上。不需要服务器，不需要数据库，不需要写代码。所有文件都是 JSON 和 Markdown 格式：你可以理解为有固定排版格式的文本文件。复制，粘贴，到任何 AI 对话窗口里就能用。

就像把复杂的图书馆系统压缩成了一叠手写的索引卡片。图书馆当然更强大，但索引卡片可以揣在兜里，走到哪儿用到哪儿。

系统解决三个问题。

**第一个：对话中断后，怎么把状态抢救回来？**

谟涅HMS 里有"谟涅Loci记忆提取协议"。名字里的 Loci 来自古希腊的记忆宫殿法：演说家为了记住长篇演讲，会把每个要点跟具体的空间位置绑定——大门是开场白，第一根柱子是第一个论点，第二根柱子是第二个论点。走遍宫殿，就回忆起整篇演讲。

谟涅HMS 用同样的思路给记忆分层：工作区、房间、厅、柜子、抽屉，五层空间，越往下越具体。

协议有两种输出模式：

- **坐标卡（轻量）**：约 170 token，一张简明的定位卡。告诉你"这次对话属于哪个项目、核心决策是什么、下一步做什么"。适合快速恢复上下文——当窗口溢出、你只是想在新窗口继续时，用这个。

- **全量台账（重型）**：完整五层空间 JSON 输出，包含每条决策、每个产出、每次测试、每个认知锚点、所有待办和警示。参考压缩比 85-95%。适合重要对话的完整归档——当项目节点、研究讨论、设计评审结束后，你想把整段记忆完整存下来时，用这个。

日常使用优先坐标卡。全量台账留给值得存的东西。两个模式由同一个协议处理，你只需要说"出坐标卡"或"出全量台账"。

**第二个：记忆越攒越多，怎么整理？**

用一段时间，你会攒下好几张记忆卡。有些信息重复了，有些过时了，有些之间有矛盾。这时候需要——"Dream认知维护协议"。

Dream 名字来自早期跨窗口记忆提取实验的代号，后来演变成了"项目级认知资产蒸馏"的意思。

它同样有两种输出模式：

- **索引更新**：把旧记忆和新提取的台账合并，更新你的长期记忆索引（5 型分类：AI工作模型 / 服务对象 / 纪律红线 / 当前任务 / 技术资产）。每次协议②运行后，你的记忆索引就是最新的。

- **认知蒸馏**：当你有跨窗口、跨项目的多张记忆卡时，把它们一起丢给协议②，它能做跨卡的认知提炼——找出散落在不同对话里的共同模式，建立认知演进时间线，甚至跨项目识别重复踩的坑。这步是可选的，只在你想做阶段性总结时做。
![Uploading 谟涅7.jpg…]()


**第三个：需要历史记忆的时候，怎么找回来？**

系统里有个文件 `Recall.md` 和 `Memory-Mode.md`——使用手册和空白模版。前者告诉你如何把之前存下来的记忆卡粘贴到新窗口里恢复上下文，后者给你一个可填写的记忆索引骨架。手动操作，复制粘贴。

## 重要：什么时候用它，什么时候别用它

谟涅HMS **不是**让你每天每段对话都跑的例行工具。它的真实定位是：

**✓ 应该用：**
- 对话窗口提示"上下文过长"，AI 开始忘记早期内容
- 平台限额（KIMI/GPT/Claude 等额度耗尽）
- 需要把工作从窗口 A 迁移到窗口 B 继续
- 对话轮次超过 50 轮，AI 开始飘忽不定
- 重要项目节点或研究讨论——值得完整归档

**✗ 不应该用：**
- 每次正常对话结束后都跑一遍（你会累、协议会贬值）
- 团队协作（一个人维护自己的记忆还行，两个人以上成本失控）
- 企业级知识库（50 张卡片的临界点眨眼就到）
- 需要自动语义召回（系统没有搜索引擎，记忆恢复是手动的）

说清楚谟涅HMS 能做什么之后，更得说清楚它不能做什么。

它最适合一个人跟 AI 长期协作的场景。你是写作者，跟 AI 讨论了很多次书稿；或者你是开发者，跟 AI 一起打磨某个项目；或者你只是想要长期的 AI 对话伙伴，让它慢慢了解你的思维方式。**这些场景下，你不是每次都需要它——但当需要时，它就在那里。**

记忆卡片不会太多。10 张以下体验最好，每次找东西，翻两下就找到了。10 到 50 张之间也还行，但需要用 Dream 协议定期做整理，不然卡片堆多了，找起来开始费劲。

**50 张是个临界点。** 在这之前，人工翻卡片的成本是可控的，你扫一眼就知道该找哪张。超过 50 张之后，人工搜索的成本开始吃掉协议带来的好处——你花在"找记忆"上的时间，可能比"用记忆"的时间还长。

这不是设计缺陷，是"零基础设施"这条路的天然边界。你选择了不需要服务器、不需要数据库、复制粘贴就能用，代价就是当数据量大到一定程度，纯文本索引扛不住。到那时候，`Recall.md` 里写得很坦白：建议部署 HMS 学术项目作为后端——当你需要的不再是索引卡片而是图书馆的时候，就该去盖一座图书馆了。

## 快速开始

```
1. 当你的聊天窗口即将溢出或对话已中断：
   打开新窗口 → 粘贴 谟涅Loci记忆提取协议.json
   → 在此窗口粘贴原始对话文本
   → 说"出坐标卡"（日常恢复）或"出全量台账"（完整归档）

2. 把输出的记忆卡保存到本地（谟涅台账_YYYYMMDD_主题.json）

3. 需要恢复上下文时：
   打开新窗口 → 粘贴记忆卡
   → 说"请读取以上记忆，确认我已恢复上下文"

4. （可选）积累多张记忆卡后：
   粘贴 Dream认知维护协议.json + 多张记忆卡
   → 说"更新索引"或"做认知蒸馏"
```

完整的系统架构和使用说明见 `docs/系统架构.md`。

## 关于协议文件格式

协议以 JSON 格式提供，目的是让 AI 模型精确解析结构（字段名、值类型、规则数组）。使用方式就是复制粘贴到对话窗口——不需要任何专有工具。如需验证 JSON 语法，可用任意 JSON 验证器检查协议正文（`{ }` 部分），末尾的协议结束标记和版本号为 LIB-204 规范要求，不影响 AI 读取。

说到底，谟涅HMS 解决的很朴素的问题：你跟 AI 聊了那么多有价值的东西，关掉窗口就全没了。

它给了你一叠索引卡片和一套整理方法。不是让你每天都翻，而是当你的抽屉乱到找不到东西时，有方法可循。在 50 张卡片的范围内，这套东西是好用的：轻、透明，你能看懂每一步在干什么。超过这个范围，它会诚实地告诉你：该去找更大的船了。
